1. Objects

Definition:

In Python, everything is an object. An object has:

  • Identity – memory address (id())
  • Type – class (type())
  • Value – actual data
x = 10
print(id(x))
print(type(x))
Interview line: Variables do not store values; they reference objects.

2. Classes

A class is a blueprint for creating objects. It defines:

  • Attributes (data)
  • Methods (behavior)
class Server:
    def start(self):
        print("Server started")

s = Server()
s.start()
Interview line: Objects are instances of classes.

3. Attributes

Attributes are names associated with an object.

Types of Attributes

  • Data attributes – store state
  • Method attributes – functions bound to class
s = "python"
print(s.upper())     # method attribute
print(s.__class__)   # data attribute
dir(s)    # list all attributes
help(str) # documentation
Interview line: Attributes are accessed using dot notation.

4. Mutability

Mutability is a class-level design decision.

Immutable Types

  • int
  • str
  • tuple
  • bool

Mutable Types

  • list
  • dict
  • set
# Immutable example
s = "python"
s2 = s.upper()  # new object created
# Mutable example
lst = [1, 2, 3]
lst.append(4)  # same object modified
Interview line: Immutable objects create new instances; mutable objects change in place.

Final Interview Summary

  • Everything in Python is an object
  • Variables reference objects
  • Classes define behavior
  • Attributes represent state and actions
  • Mutability depends on class design
